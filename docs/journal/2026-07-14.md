## How L0 honors the hard architecture rules
- **Credential scoping by construction:** each service is handed ONLY its own DB URL. agent-service gets AGENT_DATABASE_URL + RATES_SERVICE_URL + BOOKING_SERVICE_URL (REST) and NEVER rates/booking DB creds. Verified: agent-service env carries no cross-service DB creds.
- **Network segmentation:** agent-service is NOT attached to rates_db_net, so it cannot route to rates-db. Verified at runtime: agent→rates-db TCP connect times out (unreachable), while agent→rates-service:8080/health over the shared backend net succeeds.

See docs/decisions/0001-db-internal-networks-no-host-ports.md.

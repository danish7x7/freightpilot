# Session 1 — Agent L1 fixture sanitization (recorder + committed fixtures)

**Phase/Layer:** Phase 2 / agent-phase L1 (global L5) — a follow-up test/fixture + scripts-only pass, NOT new feature code and NOT a phase exit. No `src` touched.
**Session goal:** Stop committed L1 success fixtures from carrying provider blobs the adapter never reads — in particular Gemini's encrypted `thoughtSignature`, whose base64 payload trips secret-scanners as a FALSE POSITIVE — by making the recorder sanitize success bodies down to only the fields the normalizer consumes, and reshaping the committed fixtures to the same minimal form so re-recording produces no drift.

## Shipped (test/fixture + scripts only, services/agent/)
- **`scripts/record-fixtures.ts`** — new typed `sanitizeBody(kind, body)` applied to recorded SUCCESS (2xx) bodies before writing; keeps ONLY normalizer-read fields, drops everything else:
  - **Gemini** — keeps `candidates[].content.{parts (text | functionCall{name,args}), role}`, `finishReason`, `usageMetadata.{promptTokenCount, candidatesTokenCount}`. Drops `thoughtSignature`, `functionCall.id`, `thoughtsTokenCount`, `totalTokenCount`, `promptTokensDetails`, `serviceTier`, `modelVersion`, `responseId`, `index`, `finishMessage`.
  - **OpenAI-schema (Groq/Cerebras)** — keeps `choices[].message.{role, content, tool_calls[].{id, function{name, arguments}}}`, `finish_reason`, `usage.{prompt_tokens, completion_tokens}`. Drops `id`, `object`, `model`, `index`, `logprobs`, `total_tokens`, `x_groq`, `system_fingerprint`, usage timing fields, `service_tier`.
  - **Error (non-2xx) bodies pass through untouched** — the adapter reads only their status.
- **Committed success fixtures reshaped to the same minimal form** so committed == recorder output (no drift): `gemini/{text,toolcall}`, `groq/{text,toolcall}`, `cerebras/{text,toolcall}`. The 429 fixtures were left as-is.

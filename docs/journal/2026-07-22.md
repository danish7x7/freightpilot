## Key decision — Option A scope split (land the seam, defer the prompt)
- The turn endpoint runs the **REAL loop** but with **NO system prompt** (`messages=[{role:user}]`) and **NO conversation persistence**. The loop needs a system prompt to meaningfully drive `create_booking`, but a new `prompts/v1_system.md` requires an eval run + committed scorecard + eval-auditor gate (CLAUDE.md "prompt files are code") — which D14 explicitly excluded. So the wiring seam landed now; the live-LLM system prompt + eval suite are a **separate L5 prompt PR**. This is exactly why eval-auditor was correctly NOT in the review chain this session. See ADR-0010's "Revisit when."

## Learned
- LEARNING.md appended ×3: (1) **TS/Node/Fastify** — TanStack Query `refetchInterval` returning `false` does NOT trigger a re-render, so a deadline-based "stop spinning" needs an explicit `setTimeout(setPollExhausted)` to force the terminal render; found via Playwright `clock.fastForward`. (2) **Evals & Testing** — a comma inside an OpenAPI YAML FLOW-mapping scalar crashes spectral/nimma with a non-obvious `Cannot read properties of null (reading 'enum')`; diagnosed by bisecting (stock ruleset also crashed → not our rules → bisect schemas → an inline flow description), fixed with block style. (3) **Process** — a governance gate (prompt files = code) can bisect a layer: run the real loop with no system prompt to land the wiring seam, defer the live-LLM behavior to a prompt PR that carries the eval scorecard.

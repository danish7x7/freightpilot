## Architectural framing (why this is a courier, not a feature)
- The client is the **ORCHESTRATOR** (§6.2) — the same role the agent plays in Phase 2. It is a pure **courier**: it forwards rates-owned data it was handed, asserts nothing itself, does no rate math (§2.2), and never enforces booking transitions (§2.4 — the server's 409 is authoritative).
- This is exactly why PR-A had to add `lane_id` to the rates contract first: a courier can't forward what rates didn't emit.

# Session 2 — Agent L1: provider-agnostic LLM adapter

**Phase/Layer:** Phase 2 (opening) / agent-phase L1 = global L5 (MASTER_PLAN §6.1). First agent-service code: the provider-agnostic LLM layer that L2's tool loop will sit on. NOT a phase exit.
**Session goal:** Build the LLM adapter — `LlmProvider` interface + normalized shapes, GeminiProvider + one reused OpenAiCompatProvider (Groq/Cerebras), an `LLM_CHAIN` router with allowlist-only fallback, a single-source error classifier, token-bucket pacing, zod env config, and a record/replay harness that mocks at the HTTP boundary — all under `services/agent/`.

## Shipped (agent-service, all under services/agent/)
- **`src/llm/types.ts`** — `LlmProvider` interface + normalized message / tool-call / usage shapes.
- **GeminiProvider (native `generateContent`)** and **ONE `OpenAiCompatProvider` reused for BOTH Groq and Cerebras** (same class, different pinned base URL + model + key from config). Plain Node 22 global `fetch` — NO `openai` / `@google/genai` SDKs (ADR-0006): keeps HTTP-boundary replay natural + shrinks attack surface.
- **`src/llm/router.ts`** — `LLM_CHAIN` router: tries providers in order, falls back ONLY on a retryable allowlist (429/5xx/timeout/network); every other 4xx and any malformed 200 is rethrown as a typed bug, never masked by fallthrough. All exhausted → `LlmChainExhaustedError`.
- **`src/llm/errors.ts`** — error classifier as single source of truth: the `RETRYABLE` set drives BOTH `LlmError.retryable` AND the router's fallback decision, so "what we classify" and "what we retry" cannot drift.
- **`src/llm/tokenBucket.ts`** — per-provider token-bucket pacing, config-driven RPM, `Date.now`-based so fake timers drive it.
- **`src/llm/config.ts`** — zod-validated env; `LLM_CHAIN` parsed as `provider:model` pairs; base URLs PINNED in a code registry (only model/key/RPM from env → no SSRF / host injection).
- **Record/replay harness** — undici `MockAgent` mocks at the HTTP boundary (NOT the `LlmProvider` seam), so the real normalization + classifier + router run; `disableNetConnect()` guarantees zero live CI calls. Env-gated recorder `scripts/record-fixtures.ts` (`RECORD_FIXTURES=1`) lives OUTSIDE the vitest glob.
- **`services/agent/.env.example`** tracked (placeholder keys only); **`services/agent/.env`** gitignored.
